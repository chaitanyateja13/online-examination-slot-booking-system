# Contributing Guide

Thank you for your interest in contributing to the Online Examination Slot Booking System.

---

## Getting Started

1. Fork the repository.
2. Clone your fork.
3. Create a new feature branch.
4. Make your changes.
5. Test your changes.
6. Commit with a meaningful message.
7. Push your branch.
8. Open a Pull Request.

---

## Coding Standards

- Follow Python PEP 8 guidelines.
- Keep code clean and readable.
- Use meaningful variable and function names.
- Add comments where necessary.
- Reuse existing components whenever possible.

---

## Documentation

If your contribution changes functionality, update the relevant documentation.

---

## Reporting Issues

Before opening a new issue:

- Search existing issues.
- Provide clear reproduction steps.
- Include screenshots if applicable.

---

## Pull Requests

Please ensure:

- The project builds successfully.
- Documentation is updated.
- No unnecessary files are included.
- Commit messages are meaningful.

---

## Community

Be respectful and follow the project's Code of Conduct.
